# Purpose

This document defines the navigation structure of Travel Cockpit.

The objective is to minimise the number of taps required to perform common travel tasks.

Navigation should always reflect the current trip and the active accommodation.

---

# Navigation Philosophy

The application should answer one simple question:

> What do I need right now?

The interface changes naturally depending on the stage of the journey.

---

# Bottom Navigation

Travel Cockpit uses five primary navigation items.

## Home

The central dashboard.

Displays:

- Current accommodation
- Weather
- Planned activities
- AI suggestions
- Next accommodation
- Tickets
- Energy information

---

## Map

Interactive map.

Displays:

- Accommodations
- Activities
- Restaurants
- Cafés
- Energy Points
- Route between accommodations

Filters allow the traveller to show only relevant information.

---

## Planning

Displays only planned activities.

Ideas are not shown here.

The traveller can:

- Reorder activities
- Mark activities as completed
- Start navigation

---

## Discover

Shows everything not yet planned.

Examples:

- AI Suggestions
- Activities
- Restaurants
- Cafés
- Scenic viewpoints
- Museums
- Walks
- Rain alternatives

The traveller chooses what to add.

---

## Trip

Trip management.

Contains:

- Trip details
- Accommodations
- Tickets
- Travel companions
- Settings
- Export
- Offline status

---

---

# Quick Actions

Important actions are always accessible.

Examples:

- Start Google Maps
- Open Komoot
- AI Suggestions
- Add Activity
- Search Nearby
- Find Energy Point

---

# Navigation Rules

Navigation should never require more than three taps.

Important actions should never require more than one tap.

The traveller should never wonder where to find information.

Everything should have one logical location.

---

# Design Goals

The application should feel calm.

No unnecessary menus.

No duplicated information.

No hidden functionality.

Everything should be discoverable.
